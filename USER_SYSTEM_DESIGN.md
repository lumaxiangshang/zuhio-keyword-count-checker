## 🎯 用户体系设计

### 1. 用户数据模型

```typescript
interface User {
  // Firebase 提供的基础信息
  uid: string;              // 唯一用户 ID
  email: string;            // 邮箱
  displayName: string;      // 显示名称
  photoURL: string;         // 头像
  emailVerified: boolean;   // 邮箱验证状态
  createdAt: number;        // 注册时间戳
  
  // Zuhio 扩展信息
  plan: 'free' | 'pro' | 'enterprise';  // 订阅计划
  credits: number;          // 剩余额度（免费用户每日限制）
  usageCount: number;       // 今日使用次数
  lastUsedAt: number;       // 最后使用时间
  preferences: {
    language: 'en' | 'zh' | 'fr' | 'de';  // 语言偏好
    theme: 'light' | 'dark' | 'system';   // 主题偏好
    defaultKeyword?: string;               // 默认关键词
  };
  history: AnalysisRecord[];  // 分析历史记录
}
```

## 🔐 认证流程

```
1. 用户点击 "Sign in with Google"
         ↓
2. Firebase Authentication 处理登录
         ↓
3. 获取用户信息 (uid, email, displayName, photoURL)
         ↓
4. 检查是否为新用户
         ↓
5. 新用户 → 创建用户记录 (Firestore)
   老用户 → 更新最后登录时间
         ↓
6. 跳转到个人中心 /dashboard
```

## 💾 数据存储方案

### Firebase Firestore 集合

```
users/                    # 用户集合
  └── {uid}/
      ├── profile         # 个人资料
      ├── preferences     # 偏好设置
      └── usage           # 使用统计

analyses/                 # 分析记录集合
  └── {uid}/
      └── {analysisId}/
          ├── content     # 分析内容
          ├── results     # 分析结果
          └── createdAt   # 创建时间

subscriptions/            # 订阅集合
  └── {uid}/
      ├── plan            # 当前计划
      ├── status          # 状态
      └── billing         # 账单信息
```
